# 7.1 渠道接入概述

---

前面六章，你学会了用 OpenClaw 在电脑上帮你做事。但有一个问题：每次都要打开浏览器或者终端才能跟它对话。

如果能让它**在你常用的 IM 工具里待命**呢？

- 飞书群里@它，让它查数据
- 钉钉群里@它，让它回复客户
- QQ 群里@它，让它总结文章

**这一节，我们用10分钟，了解渠道接入的基本概念，选择适合你的渠道。**

---

## 场景：同一个 AI，在不同的群里待命

想象一下：

> 你正在开会，飞书群里同事问数据。你@机器人，30 秒后报表发了出来。
>
> 会议结束，钉钉响了，客户催进度。你@机器人查项目状态，直接回复。
>
> 晚上在家，QQ 群里朋友问周末去哪玩。你@机器人搜攻略，转发到群里。

**这就是渠道接入的意义**——让 AI 助手在你常用的 IM 工具里随时待命。

---

## OpenClaw 支持哪些渠道

| 渠道 | 接入方式 | 适合谁 |
|------|----------|--------|
| **飞书** | 官方插件 | 企业用户首选，功能完整 |
| **企业微信** | 社区方案 | 已有企微生态的团队 |
| **钉钉** | 官方插件 | 阿里系用户 |
| **QQ** | 官方插件 | 个人用户，门槛低 |
| **微信** | 第三方方案 | ⚠️ 有封号风险，需谨慎 |

此外还支持 Telegram、Discord、Slack 等海外渠道，以及 WebChat 网页端。

---

## 渠道接入的原理

OpenClaw 通过 **Gateway 网关** 连接各个 IM 渠道：

```
用户消息 → IM 平台 → Gateway 网关 → OpenClaw Agent → 处理回复 → 用户
```

关键点：
- **每个渠道独立配置**：飞书不影响钉钉
- **统一会话管理**：不同渠道共享同一个 Agent 的记忆
- **权限可控**：可设置谁能在哪个渠道使用

---

## 怎么选渠道

**企业用户**：
- 用飞书 → 选飞书渠道
- 用钉钉 → 选钉钉渠道
- 用企微 → 选企微渠道

**个人用户**：
- 想要稳定 → 选 QQ
- 想要功能完整 → 选 Telegram
- 想要零配置 → 选 WebChat（浏览器直接用）

⚠️ **微信个人号接入存在封号风险**，不建议主力账号使用。

---

## 试一试：查看当前渠道配置

在终端运行：

> ```
> openclaw channels list
> ```

如果显示空列表，说明还没有配置任何渠道。

---

## 配置渠道的基本流程

无论选哪个渠道，基本流程都是：

1. **安装对应插件**
2. **在 IM 平台创建机器人**，获取凭证
3. **配置权限**，让机器人能收发消息
4. **在 OpenClaw 中添加配置**
5. **启动 Gateway**，测试连接

---

## 安全提醒

- **凭证保护**：App ID、Secret 不要泄露，不要上传到公开仓库
- **权限最小化**：只给必要的权限，群组中默认需要 @提及才响应
- **风控规避**：微信个人号建议用小号测试，不要用于群发/营销
