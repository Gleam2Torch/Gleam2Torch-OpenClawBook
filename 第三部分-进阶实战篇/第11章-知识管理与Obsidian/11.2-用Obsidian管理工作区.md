# 11.2 用 Obsidian 管理工作区

---

你已经知道为什么选 Obsidian。

现在来实战：把工作区变成 Obsidian 仓库，开始可视化管理 AI 的记忆。

**这一节，我们用 20 分钟，完成工作区的可视化管理。**

---

## 准备工作

开始前，确认你有：

- [x] Obsidian 已安装
- [x] OpenClaw 工作区已存在（运行过至少一次）
- [x] 约 20 分钟时间

---

## 第一步：打开工作区

### 1.1 启动 Obsidian

打开 Obsidian，你会看到一个欢迎界面。

### 1.2 选择"打开本地仓库"

点击 **"打开本地仓库"** 或 **"Open folder as vault"**。

### 1.3 导航到工作区路径

| 系统 | 路径 |
|------|------|
| macOS | `/Users/你的用户名/.openclaw/workspace` |
| Linux | `/home/你的用户名/.openclaw/workspace` |
| Windows | `C:\Users\你的用户名\.openclaw\workspace` |

> ⚠️ **注意**: `~` 是用户主目录的缩写。在 Obsidian 的文件选择器中，你可能需要按 `Cmd+Shift+G` (macOS) 或直接输入完整路径。

### 1.4 命名仓库

给仓库起个名字，比如"OpenClaw 大脑"或"AI 工作区"。

### 1.5 知道你看到了什么

Obsidian 打开后，左侧文件列表会显示:

```
OpenClaw 大脑
├── AGENTS.md        # AI 的工作准则
├── SOUL.md          # AI 的性格设定
├── USER.md          # 你的个人偏好
├── MEMORY.md        # 长期记忆
├── HEARTBEAT.md     # 定时任务清单
├── TOOLS.md         # 工具配置
├── IDENTITY.md      # 身份定义
└── memory/          # 每日日志目录
    ├── 2026-03-08.md
    ├── 2026-03-09.md
    └── 2026-03-10.md
```

**这就是你的 AI 助手的"大脑"——全部可见了。**

## 第二步:理解核心文件

### 2.1 MEMORY.md — 长期记忆

这是最重要的文件。点击打开，你会看到类似这样的内容:

```markdown
# 长期记忆

## 用户偏好
- 喜欢简洁的回复，不要废话
- 用中文交流
- 代码块要加语法高亮

## 重要客户
- [[客户张三]] - 电商项目，偏好周报形式
- [[客户李四]] - 咨询项目，需要详细文档

## 项目信息
- #项目/电商 - 预计3月上线
- #项目/官网 - 已完成设计阶段
```

**Obsidian 会自动识别:**
- `[[客户张三]]` → 双向链接(蓝色高亮，可点击)
- `#项目/电商` → 标签(可点击筛选)

### 2.2 SOUL.md — AI 性格设定

这个文件定义了 AI 的"人格"。

```markdown
# 性格设定

## 基本性格
- 专业、高效、友好
- 主动思考，不只回答问题

## 沟通风格
- 简洁明了，避免冗长
- 适当使用列表和表格
- 重要信息用引用框突出

## 安全边界
- 不执行危险操作
- 不泄露敏感信息
- 遇到不确定的情况主动询问
```

**你可以直接编辑这个文件来改变 AI 的性格。**

### 2.3 HEARTBEAT.md — 定时任务

这是 AI 的每日检查清单:

```markdown
# 心跳检查

## 每日例行 (每天 9:00)
- [ ] 检查邮件并汇总
- [ ] 查看今日日程
- [ ] 汇总 overnight 新闻

## 本周重点
- 完成电商项目文档
- 跟进客户张三的需求
```

**Obsidian 会把 `[ ]` 渲染成可点击的复选框。** 你可以直接勾选，AI 下次会看到更新。

### 2.4 memory/ 目录 — 每日日志

这里存放每天的对话记录:

```markdown
# 2026-03-10

## 今日对话摘要
- 用户让我帮忙写周报
- 确认了客户张三的需求变更
- 讨论了新项目的排期

## 新学到的
- 用户喜欢更简洁的列表格式
- 客户张三的项目延期到3月底
```

**特点:** 仅追加，不修改;自动创建;记录短期信息。

## 第三步:查看反向链接

### 3.1 什么是反向链接

当你在 `MEMORY.md` 中写了 `[[客户张三]]`，Obsidian 会在 `客户张三.md` 的右侧面板显示:

> ```
> 链入此笔记:
> - MEMORY.md: "...重要客户 - [[客户张三]] - 电商项目..."
> ```

**这让你一眼看到:谁引用了这个笔记。**

### 3.2 打开反向链接面板

1。 点击右侧边栏的"链接"图标
2。 或使用快捷键 `Cmd/Ctrl + Shift + L`

### 3.3 实战:查看客户被引用的次数

1。 点击 `MEMORY.md`
2。 点击 `[[客户张三]]` 跳转到客户笔记
3。 查看右侧面板，看到所有引用这个客户的地方

## 第四步:编辑记忆文件

### 4.1 直接编辑

在 Obsidian 里，编辑 Markdown 文件和普通文本编辑器一样:

1。 双击文件打开
2。 直接修改内容
3. `Cmd/Ctrl + S` 保存(或自动保存)

### 4.2 场景:修改用户偏好

原来 AI 记的是:

```markdown
- 喜欢简洁的回复
```

你想改成更具体:

```markdown
- 喜欢简洁的回复，每段不超过3行
- 多用列表，少用长段落
- 重要信息用 > 引用框突出
```

**操作:**
1。 在 Obsidian 里打开 `MEMORY.md`
2。 找到"用户偏好"部分
3。 直接编辑
4。 保存

**效果:**

> ```
> 用户:帮我写个周报
>
> AI:收到。根据你的偏好，我会用简洁格式，
> 多用列表展示进展。
> ```

### 4.3 场景:创建客户档案

1。 在 Obsidian 点击"新建笔记"
2。 命名为 `客户张三`
3。 写入内容:

```markdown
# 客户张三

## 基本信息
- 公司:某某科技
- 行业:电商
- 规模:50人

## 联系方式
- 邮箱:zhangsan@example.com
- 电话:138xxxx8888

## 项目历史
- [[项目/电商平台]] - 进行中

## 偏好和特点
- 喜欢简洁汇报，关注数据
- 周五下午开会

## 重要日期
- 首次接触: 2026-01-15
- 合同到期: 2026-12-31
```

4。 回到 `MEMORY.md`，把"客户张三"改成 `[[客户张三]]`

**现在点击链接，就能跳转到详细档案。**

## 第五步:实时同步测试

### 5.1 验证双向同步

**测试 1: Obsidian 改，AI 读**

1。 在 Obsidian 里修改 `SOUL.md`，添加:
   ```markdown
   ## 新增设定
   - 每次回复开头加一个 emoji
   ```
2。 保存
3。 去 OpenClaw 发消息测试
4. AI 的回复开头应该有 emoji

**测试 2: AI 写，Obsidian 看**

1。 告诉 OpenClaw:
   > ```
   > 记住:客户王五的新项目下周启动。
   > ```
2。 切换到 Obsidian
3。 打开 `memory/今天的日期.md`
4。 看到新记录已经出现

### 5.2 自动刷新

Obsidian 默认会自动检测文件变化并刷新显示。如果没有自动刷新:

1。 设置 → 编辑器 → 自动刷新
2。 或按 `Cmd/Ctrl + R` 手动刷新

## 故障排查

    |
| 问题 | 检查步骤 |
|------|----------|
| 文件不显示 | 确认路径是 `~/.openclaw/workspace` 不是 `~/.openclaw` |
| 中文乱码 | 检查文件编码是否为 UTF-8 |
| 修改不生效 | 确认文件已保存;检查 OpenClaw 是否在运行 |
| 链接不跳转 | 确认目标笔记存在;检查 `[[ ]]` 语法 |
| 反向链接为空 | 确认其他笔记中有 `[[当前笔记名]]` |
| 图谱为空 | 确认笔记之间有双向链接 |
