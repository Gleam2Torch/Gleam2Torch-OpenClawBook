# 9.1 什么是Skills

---

前面八章，你学会了渠道接入、多 Agent 协作。但有一个问题：你的 AI 助手只会基础对话，不能搜索、不能发邮件、不能操作工具。

想象一下这个场景：

> 你问 OpenClaw：“帮我查一下今天北京的天气”
>
> OpenClaw 回复：“抱歉，我无法访问互联网，无法查询天气。”
>
> 你又问：“帮我发一封邮件给张总”
>
> OpenClaw 回复：“我没有邮件发送能力。”
>
> 你有点失望——这 AI 怎么什么都不会？

**这不是 AI 的问题，而是它没有“技能”。**

就像新员工入职时只会说话，需要培训才能干活。Skills 系统就是 OpenClaw 的“培训体系”。

**这一节，我们用10分钟，了解 Skills 的基本概念，知道如何查看已加载的技能。**

---

## Skills 是什么

Skills（技能）是教 OpenClaw 如何使用工具的说明书。

每个 Skill 是一个文件夹，里面至少包含一个 `SKILL.md` 文件。这个文件用简单的格式告诉 AI：

- 这个技能是做什么的
- 什么时候该用它
- 怎么用

举个例子，一个天气查询 Skill 的 `SKILL.md` 可能是这样的：

```markdown
---
name: 天气助手
description: 获取当前天气及未来天气预报
---

获取天气信息:
1. 调用当地天气信息获取API：XXX
2. 解析结果
3. 友好地展示结果
```

就这么简单。OpenClaw 读到这个文件，就“学会”了查天气。

---

## Skills 与原生工具的区别

OpenClaw 内置了一些原生工具（如 `bash`、`browser`、`edit`），这些是所有智能体都能用的基础能力。

Skills 是在这些原生工具之上的**封装和扩展**：

| 对比项 | 原生工具 | Skills |
|--------|----------|--------|
| 使用方式 | 通用、底层 | 场景化、高层 |
| 学习成本 | 需要告诉 AI 具体步骤 | 开箱即用 |
| 功能范围 | 基础操作 | 特定任务（发邮件、查航班、管理日程等）|
| 来源 | 内置 | 官方/社区/自己编写 |

打个比方：

- 原生工具像“手脚”——能打字、能点击、能执行命令
- Skills 像“职业技能”——会写周报、会订机票、会整理论文

## ClawHub 技能市场

OpenClaw 官方维护了一个 Skills 注册中心，叫 **ClawHub**（https://clawhub.com）。

截至 2026 年初，ClawHub 上已经汇聚了 **700+ 个 Skills**，覆盖：

- **搜索与研究**：联网搜索、论文整理、竞品分析
- **办公自动化**：邮件管理、日程安排、文档处理
- **开发辅助**：GitHub 操作、代码审查、自动化部署
- **生活助手**：天气查询、航班预订、记账管理
- **社交媒体**：Twitter/X 发帖、飞书集成、Discord 机器人

GitHub 上还有一个社区整理的 awesome-openclaw-skills 仓库，收录了超过 5000 个 Skills 的精选清单。

## 试一试：查看已加载的 Skills

打开 OpenClaw 对话，输入：

> ```
> 查看当前已加载的 skills
> ```

或者使用 CLI：

```bash
openclaw skills list
```

你会看到类似这样的输出：

```
已加载 Skills:
- browser (内置) - 浏览器自动化
- bash (内置) - 执行 shell 命令
- tavily-search - 联网搜索
- github - GitHub 集成
```

每个 Skill 后面会标注来源：内置、本地安装，还是工作区专属。
