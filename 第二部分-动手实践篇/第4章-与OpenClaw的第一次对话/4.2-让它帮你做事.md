# 4.2 让它帮你做事

---

上一节，你定义了它的身份，也介绍了你自己。

现在你可能会想：**它到底能帮我做什么？**

其实，OpenClaw的能力很简单——就四件工具。但正是这四件工具，组合起来几乎什么都能做。

这一节，我们用5分钟，试一遍它的基本能力。

---

## 它的工具箱

OpenClaw只有四件基础工具：

| 工具 | 比喻 | 能干什么 |
|------|------|----------|
| **Read** | 眼睛 | 看文件内容 |
| **Write** | 笔 | 创建新文件 |
| **Edit** | 橡皮+笔 | 修改已有文件 |
| **Bash** | 双手 | 执行命令 |

就这四个。

没有预先写好的「日历模块」，没有「语音处理插件」——什么都没有。

**但正是这种「什么都没有」，反而让它什么都能做。**

---

## 试一试：让它「看」文件

最简单的，让它帮你看看文件里写了什么。

> ```
> 帮我看一下 readme.txt 里写了什么
> ```

它会用 Read 工具读取文件，然后把内容告诉你：

> 已读取 readme.txt，内容如下：
>
> 「欢迎使用 OpenClaw」
>
> 需要帮你做什么处理吗？

---

## 试一试：让它「找」文件

不知道文件在哪？让它帮你搜。

> ```
> 帮我找一下项目里所有的 pdf 文件
> ```

它会用 Glob 工具搜索，把路径列出来：

> 找到 3 个 PDF 文件：
> 1. docs/manual.pdf
> 2. docs/guide.pdf
> 3. downloads/report.pdf
>
> 需要查看某个文件的内容吗？

---

## 试一试：让它「写」文件

不只是看，它还能帮你写。

> ```
> 帮我创建一个 todo.md，写上今天的待办：
> 1. 回复客户邮件
> 2. 写产品文档
> 3. 测试新功能
> ```

它会用 Write 工具创建文件，内容就是你说的那些。

想改已有的文件？用 Edit：

> ```
> 帮我把 readme.txt 里的「Hello World」改成「你好世界」
> ```

---

## 试一试：让它「动手」

最厉害的是 Bash 工具——它可以执行任何命令。

> ```
> 帮我看一下当前目录有什么文件
> ```

它会执行 `ls -la`，把结果告诉你：

> 当前目录内容：
> - readme.txt
> - config.yaml
> - data/
>
> 共 3 个项目。

---

## 怎么跟它说话？

就像跟一个新同事说话一样。直接说你想让它做什么就行：

| 你想做什么 | 你可以怎么说 |
|-----------|-------------|
| 查看文件 | 「帮我看一下 xxx 文件」 |
| 找文件 | 「帮我找一下 xxx 文件在哪」 |
| 搜内容 | 「帮我搜一下哪个文件里有 xxx」 |
| 创建文件 | 「帮我创建 xxx 文件，内容是...」 |
| 改文件 | 「帮我把 xxx 里的 A 改成 B」 |
| 执行命令 | 「帮我运行 xxx 命令」 |

不需要学语法，不需要背指令。

**它听得懂人话。**

---

## 遇到问题怎么办？

**它说「我不知道怎么做」** → 正常，换一种方式描述，或者给它更多上下文

**它做的不是你想要的** → 直接纠正：「不对，我要的是...不是...」

**它回复很慢** → 可能是网络或模型负载问题，稍等或换模型

---

## 这一节，你做了什么

| 试了什么 | 用了什么工具 |
|----------|-------------|
| 看文件内容 | Read |
| 找文件 | Glob |
| 写文件 / 改文件 | Write / Edit |
| 执行命令 | Bash |

这四件工具是 OpenClaw 的基本能力。后面所有的高级功能，都是这四件的组合。
