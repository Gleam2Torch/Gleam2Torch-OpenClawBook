# 9.2 从 ClawHub 安装技能

---

上一节你知道了 Skills 是什么。但光知道概念没用，**你得会装、会用**。

想象一下这个场景：

> 你刚配置好 OpenClaw，想让它帮你查一下竞品的最新动态。
>
> 你问：“帮我搜索某某公司的最新产品发布”
>
> OpenClaw 回复：“抱歉，我无法访问互联网，无法进行搜索。”
>
> 你有点失望——这 AI 怎么连网都上不了？

**不是 AI 不行，是没装 Skills。**

这就像你买了一台新电脑，但没装浏览器、没装 Office，当然什么也干不了。

ClawHub 是 OpenClaw 的官方技能市场，拥有 700+ 社区贡献的技能包。**这一节，我们用15分钟，学会从 ClawHub 安装技能，让你的 OpenClaw 真正“长出手脚”。**

---

## 准备工作

开始前，确认你有：

- Node.js 环境（Node 18+）
- OpenClaw 已安装并运行
- 约 15 分钟时间

---

## 第一步：安装 ClawHub CLI

ClawHub 提供了命令行工具，方便你搜索、安装、管理 Skills。

### 1.1 安装命令

```bash
npm i -g clawhub
```

或者用 pnpm：

```bash
pnpm add -g clawhub
```

### 1.2 验证安装

```bash
clawhub --version
```

看到版本号输出，说明安装成功。

---

## 第二步：搜索技能

不知道装哪个？先搜索。

### 2.1 基本搜索

```bash
clawhub search "calendar"
```

### 2.2 自然语言搜索

ClawHub 使用向量搜索，支持自然语言描述：

```bash
clawhub search "send email"
clawhub search "postgres backup"
clawhub search "flight search"
```

### 2.3 搜索结果解读

搜索结果会显示：

| 字段 | 说明 |
|------|------|
| **名称** | Skill 的显示名称 |
| **slug** | 安装标识（如 `tavily-search`） |
| **描述** | 功能简介 |
| **下载量** | 社区使用热度 |
| **作者** | 开发者信息 |

---

## 第三步：安装技能

找到想要的 Skill 后，用 slug 安装。

### 3.1 基本安装

```bash
clawhub install tavily-search
```

### 3.2 指定版本

```bash
clawhub install tavily-search --version 1.2.0
```

### 3.3 强制覆盖

如果 Skill 已存在，想重新安装：

```bash
clawhub install tavily-search --force
```

### 3.4 安装位置

默认情况下，Skills 安装到当前目录的 `./skills` 文件夹。如果配置了 OpenClaw 工作区，会安装到工作区的 skills 目录。

---

## 试一试：安装第一个 Skill

让我们安装一个实用的联网搜索技能，让 OpenClaw 能上网查资料。

### 步骤 1：搜索

```bash
clawhub search "web search"
```

从结果中找到 `tavily-search`。

### 步骤 2：安装

```bash
clawhub install tavily-search
```

### 步骤 3：重启会话

安装完成后，**启动一个新的 OpenClaw 会话**，新 Skill 才会生效。

### 步骤 4：测试

在对话中输入：

> ```
> 搜索今天的科技新闻
> ```

如果 Skill 安装成功，OpenClaw 会调用搜索工具返回结果。

## 管理已安装的技能

### 查看已安装

```bash
clawhub list
```

这会读取 `.clawhub/lock.json`，显示当前工作区安装的所有 Skills 及其版本。

### 更新单个技能

```bash
clawhub update tavily-search
```

### 更新所有技能

```bash
clawhub update --all
```

> **注意**：更新前建议查看变更日志，特别是涉及权限、凭据、命令执行的 Skills。

## 故障排查

### 安装失败

检查清单：

- [ ] Node.js 版本是否 18+：`node --version`
- [ ] npm/pnpm 是否能正常访问网络
- [ ] 是否有写入权限：检查当前目录是否可写
- [ ] 查看详细错误：`clawhub install <slug> --verbose`

### 安装后 Skill 不生效

检查清单：

- [ ] 是否重启了 OpenClaw 会话
- [ ] Skill 是否安装到正确的目录
- [ ] 运行 `clawhub list` 确认安装成功
- [ ] 检查 Skill 是否需要额外配置（如 API Key）

### 搜索无结果

检查清单：

- [ ] 网络是否正常连接
- [ ] 关键词是否准确（尝试英文关键词）
- [ ] 换个描述方式搜索（向量搜索支持自然语言）

### Skill 运行报错

检查清单：

- [ ] 查看 Skill 文档，确认所需凭据已配置
- [ ] 检查 API Key 是否有效
- [ ] 查看日志获取详细错误：`openclaw logs --follow`

## 这一节，你做了什么

| 做了什么 | 核心命令 |
|----------|----------|

- 安装前建议查看 Skill 的文档，了解需要的权限和凭据
- 不要一次性安装太多 Skills，先装 2-3 个核心需求的
