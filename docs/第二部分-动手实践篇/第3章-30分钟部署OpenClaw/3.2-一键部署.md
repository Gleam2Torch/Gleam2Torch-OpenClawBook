# 3.2 一键部署

---

准备工作完成后，接下来进行部署。

本节分为三个步骤：**安装OpenClaw** → **配置向导** → **启动Gateway**。全部完成预计需要20分钟。

---

## 第一步：安装OpenClaw

### 1.1 打开终端

- **Mac / Linux用户**：打开「终端」
- **Windows用户**：以管理员模式打开PowerShell，输入 `wsl` 进入Linux终端

### 1.2 执行安装命令

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

命令执行后，终端会显示下载和安装进度。正常情况下，10分钟左右完成。

### 1.3 确认安装成功

安装成功后，终端会显示：

```
🦞 OpenClaw installed successfully (2026.X.X)!
```

> 💡 **常见安装问题：**
> 1. **网络超时** → 切换网络或使用手机热点重试
> 2. **权限不足** → 前面加 `sudo` 再运行一次
> 3. **Node版本不对** → 确认 `node -v` 显示 v22+
> 
> 完整故障排除请参考 **附录A：故障排除指南**。

---

## 第二步：配置向导

### 2.1 启动配置向导

正常情况下，OpenClaw安装完成后会自动引导配置。如未自动引导，可手动执行：

```bash
openclaw onboard --install-daemon
```

### 2.2 配置步骤

配置向导会依次显示以下选项：

**步骤1：风险确认**

```
I understand this is powerful and inherently risky. Continue?
 Yes / No
```

选择 `Yes` 继续安装。

**步骤2：安装模式**

```
Onboarding mode
 QuickStart (Configure details later via openclaw configure.)
 Manual (Configure port, network, Tailscale, and auth options.)
```

选择 `QuickStart`，其他配置可在后续使用中逐步设置。

**步骤3：配置文件处理**

```
Config handling
 Use existing values
 Update values
 Reset
```

选择 `Use existing values` 即可。

**步骤4：选择模型**

```
Model/auth provider
 OpenAI
 Anthropic
 MiniMax
 Moonshot AI
 ...
 Skip for now
```

选择所购买的AI服务商，然后按照提示选择具体模型、粘贴API Key。

> ⚠️ **注意**：粘贴API Key时不要带空格。

**步骤5：IM通道配置**

```
Select channel (Quick Start)
 Telegram
 WhatsApp
 Discord
 ...
 Skip for now
```

暂时选择 `Skip for now`，后续可在Web UI中配置。

**步骤6：技能配置**

```
Configure skills now? (recommended)
 Yes / No
```

选择 `No`，后续可在Web UI中配置。

**步骤7：完成安装**

```
Gateway service already installed
 Restart
 Reinstall
 Skip
```

选择 `Restart` 重启服务。

### 2.3 记录Gateway Token

配置完成后，终端会显示Gateway Token链接，**请务必记录**：

```
Dashboard ready

Dashboard link (with token):
http://127.0.0.1:18789/?token=[your_gateway_token]
```

首次访问Web UI需要验证Token。

> 💡 **忘记Token怎么办？**
> 
> 执行以下命令查看：
> ```bash
> cat ~/.openclaw/openclaw.json | grep token
> ```
> 复制 `"token": "[your_gateway_token]"` 中的Token值。
> 
> 或者在浏览器打开 http://127.0.0.1:18789/ ，点击左侧 Overview，在 Gateway Access → Gateway Token 中填入Token，点击 Connect。

## 第三步：启动Gateway

### 3.1 启动服务

配置完成后，Gateway默认自动启动。如未启动，可手动执行：

```bash
openclaw gateway start
```

> 💡 **端口占用处理**：如果提示18789端口被占用，可更换端口：
> ```bash
> openclaw gateway start --port 18790
> ```

### 3.2 解锁文件操作权限（2026.3.2版本后）

OpenClaw在2026.3.2版本后，为避免对文件系统造成破坏，默认限制了文件操作能力。如需完整功能，需手动解锁：

**操作步骤：**

1. 打开 http://127.0.0.1:18789/config/

2. 点击 **All Settings**，底部选择 **Raw**

3. 在Raw JSON5配置中，找到「"tools"」部分（如无则新增），替换或添加以下内容：

```json
"tools": {
    "profile": "full",
    "exec": {
        "host": "gateway",
        "security": "full",
        "ask": "off"
    },
    "web": {
        "fetch": {
            "enabled": true
        }
    }
}
```

4. 点击 **Save** 保存，配置立即生效。

> 📖 **说明**：此配置将工具模式从 `messaging`（仅消息模式）改为 `full`（完整模式），并开放本机执行的完整权限。
