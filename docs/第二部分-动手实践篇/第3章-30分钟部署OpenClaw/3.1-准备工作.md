## 1. 一台电脑

OpenClaw需要运行在类Unix环境中。

### Mac / Linux用户

直接使用，无需额外配置。

### Windows用户

Windows需要安装 **WSL2**（Windows Subsystem for Linux 2）。

> **什么是WSL2？**
>
> WSL2是Windows的Linux子系统，让Windows可以运行Linux程序。OpenClaw基于Node.js开发，需要Linux环境才能正常运行。

**WSL2安装步骤（10分钟）：**

**步骤1：检查Windows版本**

必须满足以下条件之一：
- Windows 11 任意版本
- Windows 10 版本2004或更高（内部版本19041+）

检查方法：按 `Win + R`，输入 `winver`，回车查看版本号。

**步骤2：安装WSL2**

以管理员身份打开Windows PowerShell，执行：

```powershell
wsl --install
```

这条命令会自动安装WSL2和Linux系统（Ubuntu）。安装完成后需要重启电脑。

**步骤3：配置Ubuntu**

重启后，打开Windows PowerShell并输入 `wsl` ，系统会自动进入Linux（Ubuntu）终端。按提示设置用户名和密码。

**步骤4：启用systemd**

OpenClaw的Gateway服务需要systemd支持。在Linux（Ubuntu）终端中执行：

```bash
sudo tee /etc/wsl.conf >/dev/null <<'EOF'
[boot]
systemd=true
EOF
```

然后按 `Ctrl + D` 回到Windows PowerShell，重启WSL：

```powershell
wsl --shutdown
```

**步骤5：验证安装**

重新在Windows PowerShell界面输入 `wsl` 打开Linux（Ubuntu）终端，执行：

```bash
systemctl --user status
```

看到 `State: running` 即表示成功。

> 📚 **遇到问题？** 完整的WSL2故障排除请参考 **附录B：Windows用户专属指南**。

## 2. 一套环境

### 什么是Node.js环境？

Node.js是OpenClaw的运行环境，类似于Java程序需要JVM。

### 版本要求

OpenClaw需要 **Node.js 22.x** 或更高版本。

### 检查当前版本

打开终端，执行：

```bash
node -v
```

- 显示 `v22.x.x` 或更高 → 已满足要求，跳过安装
- 显示其他版本或 `command not found` → 需要安装

### 安装方法

**Mac用户（推荐nvm）：**

```bash
# 安装nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
source ~/.zshrc

# 安装Node.js 22
nvm install 22
nvm alias default 22

# 验证
node -v
```

**Linux用户：**

```bash
# 使用NodeSource仓库
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证
node -v
```

**Windows用户：**

使用管理员模式进入Windows PowerShell界面，输入 `wsl` 打开Linux（Ubuntu）终端，执行：

```bash
# 使用NodeSource仓库
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证
node -v
```

> 💡 **国内用户加速**：安装后执行以下命令配置npm镜像：
> ```bash
> npm config set registry https://registry.npmmirror.com
> ```

## 3. 一个大模型API Key

### 3.1 什么是API Key？

API Key是调用AI服务的身份凭证。OpenClaw需要调用AI模型来处理任务，API Key用于身份认证。

### 3.2 什么是Coding Plan？

**Coding Plan是专为编程场景设计的计费套餐。**

OpenClaw的设计原理是：每次交互都会拼接完整的上下文（包括历史对话、记忆文件、系统提示词等），因此**非常消耗token**。

| 计费方式 | 计算逻辑 | OpenClaw使用场景 |
|----------|----------|------------------|
| 按Token计费 | 上下文越长越贵 | 一次对话可能消耗几万tokens，成本高 |
| Coding Plan | 按Prompt数计费，不按Token量 | 上下文再长也按次收费，成本可控 |

**结论：使用OpenClaw，务必购买Coding Plan。**

### 3.4 选择建议

#### 国内还是国外？

**推荐优先选择国内模型**，原因如下：
- 国外API存在被墙风险，访问不稳定
- 汇率波动导致价格不稳定
- 支付渠道有限，充值不便
- 除OpenAI外，Google和Claude存在封禁风险

#### 国内模型选择

| 场景 | 推荐模型 | 原因 |
|------|----------|------|
| 综合能力最强 | GLM-5 | 国产模型综合性能最优，工具调用能力强 |
| 预算有限 | Qwen 3.5 | 价格实惠，中文理解好 |
| 长文本处理 | Kimi K2.5 | 长上下文窗口，处理长文档优秀 |
| 追求速度 | MiniMax M2.1 | 响应速度快，适合工具调用场景 |

#### 国外模型选择

如确有稳定访问渠道，且可以接受高昂的token费用，可考虑：

| 场景 | 推荐模型 | 原因 |
|------|----------|------|
| 综合能力最强 | GPT-5.4 | 全球领先，生态最完善 |
| 编程能力最强 | Claude 4.6（Opus-4.6） | 代码生成与理解能力顶尖 |
| 多模态需求 | Gemini 3.1 | 图像、视频理解能力强 |

### 3.6 获取API Key

以智谱GLM为例：

1. 访问 https://open.bigmodel.cn/
2. 注册账号（支持手机号）
3. 进入「工作台」→「API密钥管理」
4. 点击「创建API密钥」
5. 复制生成的密钥

> ⚠️ **安全提醒**：API Key相当于账户密码，不要分享给他人，不要提交到公开代码仓库。更多安全实践请参考 **附录C：隐私与安全指南**。
